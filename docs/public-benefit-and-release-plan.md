# Public Benefit And Release Plan

## Licensing Approach

The code and repository structure are intended for release under the MIT License. Documentation may also be shared under a permissive attribution model such as CC BY 4.0 if the project later separates documentation licensing from code licensing.

## Dataset Release Rules

Public dataset materials will be limited to synthetic or explicitly anonymized examples. This repository does not publish confidential evidence, customer records, audit artifacts, screenshots from real systems, or personally identifiable information.

## Public Benefit Goals

The repository is meant to provide:

- a reusable schema for integrity-focused compliance evidence research
- a transparent methodology for simulated anomaly generation
- benchmark definitions that other defensive researchers can critique or extend
- a clearly bounded example of agent-assisted integrity analysis without offensive scope

## Staged Release Plan

1. Release the repository scaffold, proposal materials, and guardrails.
2. Release the Compliance Evidence Integrity Dataset schema.
3. Release an initial set of synthetic anomaly samples.
4. Release methodology, threat model, and simulation documents.
5. Release benchmark definitions and initial results only when they can be documented with limitations and without overclaiming.

## Distribution Constraints

- no confidential or personal data
- no real customer evidence
- no secrets or environment-specific credentials
- no operational exploit content
- no claims of compliance certification or production deployment

## Review Standard

Before each public release, the repository should be reviewed for three questions:

1. Does any artifact imply stronger capability than has actually been demonstrated?
2. Does any artifact expose sensitive information or real-world evidence?
3. Does the release remain clearly defensive and useful for public scrutiny?
